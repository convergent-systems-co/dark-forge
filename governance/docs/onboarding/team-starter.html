💡 What Is This?

🤖 A Virtual Engineering Team

Drop a .ai submodule into any repo and get a full AI team: planner, coder, security reviewer, data governance reviewer, and more — all automated.

📋 Governed by Panels

Every code change goes through review panels. If panels pass, it auto-merges. If they fail, it fixes itself and tries again.

🗂️ Plans as Audit Trail

Every decision is written to .plans/. Team members can see exactly why a change was made and how it evolved.

🌐 Local or GitHub

Works offline with no GitHub remote (local mode) or fully connected with issue tracking, PRs, and auto-merge (remote mode).

📦 Before You Start

  • 🐍
    Python 3.12+ — Required for the governance policy engine.
    Download from python.org and make sure it's in your PATH.
    Verify: python --version
  • 🔧
    Git — Standard git installation.
    Verify: git --version
  • 🤖
    AI Tool (one of): GitHub Copilot CLI, Claude Code, or Cursor.
    The framework works with all three. Instructions are shared automatically via symlinks.
  • 🔑
    GitHub access to SET-Apps org — Needed to clone the submodule.
    Use SSH: git@github.com:SET-Apps/ai-submodule.git
Windows tip: For symlinks to work (so .github/copilot-instructions.md, CLAUDE.md, and .cursorrules stay in sync automatically), enable Developer Mode: Settings → System → For developers → Developer Mode → On. Otherwise the script falls back to file copies — everything still works, but you'll need to re-run init.ps1 after submodule updates.

🚀 Quick Start (Windows)

1
Create your project folder
Start a new folder for your project and initialize git.
mkdir my-project
cd my-project
git init
2
Add the .ai governance submodule
This pulls the entire governance framework into your repo as a .ai folder.
git submodule add git@github.com:SET-Apps/ai-submodule.git .ai
3
Bootstrap the framework
Run the init script. It creates symlinks for all AI tools, sets up .plans/ and .panels/ folders, copies governance workflows, and generates CODEOWNERS.
powershell -ExecutionPolicy Bypass -File .ai\init.ps1 -InstallDeps
⚡ -InstallDeps creates a Python venv and installs jsonschema + pyyaml automatically
4
Pick your language template
Copy the template for your stack and customize it. This tells the framework what conventions and policy profile to use.
# Choose one: python / node / go / csharp / react
Copy-Item .ai\templates\node\project.yaml project.yaml
Open project.yaml and set your project name. Leave everything else as-is for now.
5
Commit the setup
git add -A
git commit -m "chore: add .ai governance submodule"
6
Start the agentic loop
Open your AI tool (Copilot CLI, Claude Code, or Cursor) in this folder and type:
/startup
🤖 The framework will scan for open issues (if GitHub remote exists) or prompt you for what to build (local mode)

🔀 Local Mode vs. Remote Mode

The framework auto-detects whether you have a GitHub remote. Both modes run the full governance pipeline.

💻 Local Mode No remote needed

  • Plans written to .plans/
  • Panels run and evaluate code
  • Code generated and tested
  • No GitHub issues created
  • No PRs or auto-merge

Perfect for learning and POC. Start here.

☁️ Remote Mode Full pipeline

  • Plans written to .plans/
  • Panels run and evaluate code
  • Code generated and tested
  • GitHub issues created automatically
  • PRs created and auto-merged if panels pass

Production workflow. Add git remote add origin to activate.

📂 Understanding the .plans Folder

Every non-trivial action creates a plan file. This is your audit trail.

📄 What's in a plan?

What was asked, what was decided, what was implemented, and why. Each plan = one session or one feature.

👀 Who reads them?

New team members who need to understand why the code looks the way it does. Plans are the "how did we get here" trail.

🧹 Can we delete them?

Yes, once the team is comfortable. Thomas: "Once people get comfortable we can change the framework to stop saving plans."

📦 Where do they go?

Archived to GitHub Releases automatically when a PR merges (via plan-archival.yml workflow).

🔄 Keeping the Framework Updated

# Pull latest governance updates from SET-Apps/ai-submodule
git submodule update --remote .ai
git add .ai
git commit -m "chore: update .ai submodule"
The startup loop auto-runs this check at the start of every session. You usually don't need to do this manually.

🛟 When Things Get Stuck

SituationWhat to do
Agent stopped mid-session/startup — it resumes from the most recent checkpoint or open PR
Context is full / agent repeating itselfSay: "Write a checkpoint and stop" → /clear → "Resume from checkpoint"
Dirty git stategit add -A && git commit -m "wip: recovery" then /startup
PR stuck / CI failingSay: "Resume PR #N" — agent re-enters the review loop
Don't know where you aregit status · git branch --show-current · ls .checkpoints/
