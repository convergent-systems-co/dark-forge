# Code Quality Instructions

<!-- PHASE:code -->

Loaded for code implementation and review tasks.

## Standards

- Write idiomatic code for the target language
- Follow existing patterns in the codebase before introducing new ones
- Include error handling at system boundaries
- Write testable code — prefer dependency injection over hard-wired dependencies
- Keep functions focused on a single responsibility
- Use meaningful names that reveal intent

## Review Lens

- Correctness over cleverness
- Readability over brevity
- Consistency with codebase conventions over personal preference
- Production safety — consider concurrency, failure modes, and edge cases
