# Communication Instructions

<!-- PHASE:communication -->

Loaded for PR descriptions, issue triage, and documentation tasks.

## Formatting

- Use markdown formatting for all written output
- Link to relevant files and line numbers when discussing code
- Provide context for suggestions — state the problem before the solution

## Pull Requests

- Write clear, concise PR descriptions that explain why, not just what
- Include test plan and verification steps
- Reference the originating issue or design intent

## Issues

- Reproduce before diagnosing
- Include steps to reproduce, expected behavior, and actual behavior
- Classify severity and impact before proposing solutions
